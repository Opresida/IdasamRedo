Temos que refatorar a lógica matemática do componente GerarPdfsTab no ficheiro client/src/pages/dashboard/capacitacao.tsx.

O PROBLEMA ARQUITETURAL:
Atualmente, o sistema salva coordenadas absolutas (x e y em pixels da tela). Isto quebra o layout se a configuração for salva num monitor grande e depois o PDF for gerado ou editado num ecrã pequeno (portátil/mobile). Precisamos de implementar uma Escala Relativa Bidirecional (Percentagens). Além disso, o carregamento do PDF precisa de uma barra de progresso, e o desenho final no pdf-lib sofre de desvio de baseline.

PASSO A PASSO DE IMPLEMENTAÇÃO OBRIGATÓRIO:

1. Estado de Dimensões Dinâmicas:
Cria estados para rastrear o tamanho real do PDF e o tamanho da tela atual:

const [pdfOriginalSize, setPdfOriginalSize] = useState({ width: 842, height: 595 }); // Tamanho real A4 Paisagem
const [domContainerSize, setDomContainerSize] = useState({ width: 0, height: 0 }); // Tamanho no ecrã
const pdfWrapperRef = useRef<HTMLDivElement>(null);


Usa um ResizeObserver num useEffect acoplado ao pdfWrapperRef.current para atualizar constantemente o domContainerSize sempre que a janela for redimensionada.

2. Refatorar o Estado dos Blocos (Usar Percentagens):
Altera a estrutura do estado blocks para usar pctX e pctY (valores de 0.0 a 1.0) em vez de pixels. E baseSize para representar o tamanho real da fonte em PT no PDF final.

const [blocks, setBlocks] = useState({
  nome: { text: "{aluno}", pctX: 0.1, pctY: 0.2, baseSize: 32, font: 'Alex Brush', bold: false, italic: false },
  descricao: { text: "Certificamos {aluno}...", pctX: 0.1, pctY: 0.4, baseSize: 12, font: 'Poppins', bold: false, italic: false },
  // ... aplica pctX e pctY aos restantes
});


3. O Editor Visual (A Pré-visualização Perfeita na Tela):
No render do <Draggable>, a posição visual e o tamanho da fonte têm de ser calculados em tempo real usando regra de 3:

const visualX = block.pctX * domContainerSize.width;
const visualY = block.pctY * domContainerSize.height;
// Escala a fonte da tela para parecer proporcional ao PDF real
const scaleRatio = domContainerSize.width / pdfOriginalSize.width;
const visualFontSize = block.baseSize * scaleRatio;

<Draggable
  position={{ x: visualX, y: visualY }}
  onStop={(e, data) => {
    // Ao parar de arrastar, guarda a PERCENTAGEM e não o pixel!
    const newPctX = data.x / domContainerSize.width;
    const newPctY = data.y / domContainerSize.height;
    updateBlock(key, { pctX: newPctX, pctY: newPctY });
  }}
>
  <div style={{ fontSize: `${visualFontSize}px` }}>...</div>
</Draggable>


4. A Matemática de Geração (pdf-lib):
No handleGenerate, a matemática torna-se infalível e independente da tela do utilizador.

// Obtém o tamanho real da página do PDF processado
const { width: pdfWidth, height: pdfHeight } = page.getSize();

const finalX = block.pctX * pdfWidth;
// Fator 0.8 compensa a altura da Baseline da fonte para não ficar "flutuando" acima da linha!
const finalY = pdfHeight - (block.pctY * pdfHeight) - (block.baseSize * 0.8);

page.drawText(textoProcessado, {
  x: finalX,
  y: finalY,
  size: block.baseSize,
  font: fontSelecionada, // Assegura a extração das fontes injetadas (Alex Brush, Poppins)
  maxWidth: pdfWidth * 0.8 // Impede texto de sair da folha
});


5. Barra de Carregamento e Eventos do react-pdf:

Adiciona o estado const [loadProgress, setLoadProgress] = useState(0);.

No componente <Document>, adiciona a prop: onLoadProgress={({ loaded, total }) => setLoadProgress((loaded / total) * 100)}.

Renderiza <Progress value={loadProgress} className="w-full" /> enquanto o PDF não estiver totalmente carregado.

Na <Page onLoadSuccess={(page) => setPdfOriginalSize({ width: page.width, height: page.height })} />, captura o tamanho original do documento de fundo automaticamente.
