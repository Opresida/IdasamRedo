### PROMPT DE RECURSO: ALINHAMENTO DE TEXTO (ESQUERDA, CENTRO, DIREITA)

**Atua como um Arquiteto Front-end Sénior.**
Precisamos adicionar a funcionalidade de alinhamento de texto (Esquerda, Centro e Direita) aos blocos do nosso gerador de PDFs no arquivo `client/src/pages/dashboard/capacitacao.tsx`.

**O DESAFIO TÉCNICO (ATENÇÃO):**
A interface React resolve isso fácil com CSS `textAlign`, mas a biblioteca `pdf-lib` NÃO possui uma propriedade nativa de alinhamento para textos multilinhas no método `page.drawText()`. Para que o alinhamento funcione no PDF gerado, terás de implementar uma lógica de renderização linha a linha (Word Wrap manual).

**PASSO A PASSO DA IMPLEMENTAÇÃO:**

**1. Atualização do Estado e Interface (UI):**
- Adiciona a propriedade `align: 'left' | 'center' | 'right'` ao estado de configuração dos blocos (valor padrão: 'left').
- Importa os ícones `AlignLeft`, `AlignCenter` e `AlignRight` do `lucide-react`.
- No painel lateral de edição de blocos, adiciona um grupo de botões (ex: ToggleGroup ou um row de botões) para permitir a escolha do alinhamento (tal como temos para Bold/Italic).
- No componente `NativeDraggableBlock`, aplica visualmente o estilo `textAlign: block.align`. (Nota: para que o alinhamento se note visualmente durante o Drag, a div pode precisar de uma largura fixa correspondente ao `maxWidth` que usamos no PDF, ou o utilizador arrasta o ponto de origem e o texto expande a partir daí).

**2. A Matemática do PDF-LIB (A Geração Final):**
Na função `generateCertificatePdf`, a abordagem de usar um único `drawText` com `maxWidth` já não serve se o alinhamento for `center` ou `right`.
Substitui a chamada simples do `page.drawText` por um algoritmo que faça o seguinte para cada bloco:
- Usa (ou cria) uma função auxiliar para aplicar Word-Wrap: quebra o `resolvedText` num array de linhas baseando-se no `maxWidth` (ex: `pdfWidth * 0.8`) usando `embedFont.widthOfTextAtSize()`. Respeita também as quebras de linha manuais (`\n`).
- Faz um loop sobre essas linhas geradas.
- Para cada linha, calcula a largura exata: `const lineWidth = embedFont.widthOfTextAtSize(line, safeSize)`.
- Calcula a coordenada `x` de cada linha com base no alinhamento do bloco:
  - Se `left`: `x = finalX`
  - Se `center`: `x = finalX + (maxWidth - lineWidth) / 2`
  - Se `right`: `x = finalX + (maxWidth - lineWidth)`
- Calcula a coordenada `y` de cada linha: `y = finalY - (index * safeSize * 1.2)` (aplicando um lineHeight básico).
- Usa `page.drawText` para desenhar CADA linha individualmente nas suas respetivas coordenadas X e Y calculadas.

**REGRAS RÍGIDAS:**
- Não modifiques a lógica de Drag & Drop (`NativeDraggableBlock`) com os eventos nativos que implementámos anteriormente. Adiciona apenas a injeção do CSS visual de alinhamento.
- Garante que o cálculo de "Baseline" da primeira linha continua a existir na lógica de geração para o texto não ficar flutuando.
