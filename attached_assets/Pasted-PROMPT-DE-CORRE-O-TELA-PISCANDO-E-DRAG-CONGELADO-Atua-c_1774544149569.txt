PROMPT DE CORREÇÃO: TELA PISCANDO E DRAG CONGELADO

Atua como um Arquiteto Front-end Sénior.
A nova arquitetura de percentagens e escala bidirecional do componente GerarPdfsTab em client/src/pages/dashboard/capacitacao.tsx está a causar um loop de re-renderização (tela a piscar) e os blocos de texto não podem ser arrastados.

AS CAUSAS IDENTIFICADAS:

O ResizeObserver está num loop infinito de setState.

Os cálculos visuais de x e y no <Draggable> estão a receber 0 ou NaN durante o loading inicial, quebrando a biblioteca.

Conflitos de position no <Draggable> durante o arrasto.

APLICA IMEDIATAMENTE AS SEGUINTES CORREÇÕES:

1. Trava de Segurança no ResizeObserver (Debounce/Check):
Atualiza o teu useEffect do ResizeObserver para só atualizar o estado se as dimensões forem realmente diferentes e maiores que zero:

useEffect(() => {
  if (!pdfWrapperRef.current) return;
  const observer = new ResizeObserver((entries) => {
    const entry = entries;
    if (entry) {
      const { clientWidth, clientHeight } = entry.target as HTMLDivElement;
      if (clientWidth > 0 && clientHeight > 0) {
        setDomContainerSize(prev => {
          // Só atualiza se a diferença for maior que 2 pixels para evitar loop infinito de sub-pixels
          if (Math.abs(prev.width - clientWidth) > 2 || Math.abs(prev.height - clientHeight) > 2) {
            return { width: clientWidth, height: clientHeight };
          }
          return prev;
        });
      }
    }
  });
  observer.observe(pdfWrapperRef.current);
  return () => observer.disconnect();
}, []);


2. Renderização Condicional dos Blocos (Fim do Bug NaN):
NÃO renderizes o mapeamento de blocos (Object.entries(blocks).map...) se domContainerSize.width === 0 ou se o PDF ainda estiver a carregar.
Envolve os blocos com:

{domContainerSize.width > 0 && loadProgress === 100 && (
  Object.entries(blocks).map(([key, block]) => { ... })
)}


3. Correção do Evento <Draggable>:
Garante que o <Draggable> tem o nodeRef e usa a posição convertida corretamente. MUITO IMPORTANTE: no onStop, atualiza o estado garantindo limites entre 0.0 e 1.0.

const visualX = block.pctX * domContainerSize.width;
const visualY = block.pctY * domContainerSize.height;
// Scale ratio da font
const visualFontSize = block.baseSize * (domContainerSize.width / pdfOriginalSize.width);
