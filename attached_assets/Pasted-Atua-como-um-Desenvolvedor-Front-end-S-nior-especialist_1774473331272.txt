Atua como um Desenvolvedor Front-end Sénior especialista em React, TypeScript e na criação de interfaces visuais interativas (Drag & Drop).

Estamos a trabalhar no ficheiro client/src/pages/dashboard/capacitacao.tsx, especificamente na funcionalidade da aba "Gerar PDFs" (componente GerarPdfsTab).

O OBJETIVO ATUAL:
Precisamos de evoluir o Editor Visual "Arrastar e Largar" (Drag & Drop) do PDF para uma experiência fluida e de nível profissional. O utilizador deve poder editar o texto de cada bloco (usando variáveis como {aluno}, {curso}), escolher o tamanho da fonte, e aplicar estilos (Negrito e Itálico). Deve também existir uma barra de carregamento fluida enquanto o PDF é processado pelo react-pdf.

PASSO A PASSO DA IMPLEMENTAÇÃO:

1. Dependências e Fontes:

Garante que react-draggable, react-pdf, pdf-lib e @pdf-lib/fontkit estão instalados.

Importa os ícones Bold, Italic, Type, Loader2 do lucide-react.

2. Estrutura de Estado Robusta (Substituir o coords simples):
Cria um estado blocks que guarda as propriedades de cada elemento de texto. Exemplo de estrutura:

const [blocks, setBlocks] = useState({
  nome: { text: "{aluno}", x: 50, y: 150, size: 32, font: 'Alex Brush', bold: false, italic: false },
  descricao: { text: "O Instituto IDASAM certifica que {aluno} concluiu o curso de {curso} com carga horária de {carga}h.", x: 50, y: 300, size: 12, font: 'Poppins', bold: false, italic: false },
  ementa: { text: "{ementa}", x: 50, y: 700, size: 10, font: 'Poppins', bold: false, italic: false },
  instrutor: { text: "{instrutor}", x: 150, y: 150, size: 12, font: 'Poppins', bold: true, italic: false },
  emissao: { text: "{data}", x: 150, y: 120, size: 12, font: 'Poppins', bold: false, italic: false },
  codigo: { text: "{codigo}", x: 250, y: 90, size: 10, font: 'Poppins', bold: false, italic: false }
});


3. Layout Dividido (Split Screen):

Lado Esquerdo (Painel de Controlo): - Seleção do Curso e Upload do PDF.

Se o PDF for carregado, mostra uma lista de Cartões (Cards) para cada bloco de texto. Em cada cartão, o utilizador pode:

Editar o campo text (num <Textarea> ou <Input>).

Alterar o size (tamanho da fonte).

Clicar em botões Toggle para bold e italic.

Lado Direito (Preview Visual): - Renderiza o <Document> do react-pdf.

Barra de Carregamento: Usa a prop onLoadProgress para capturar a percentagem e exibir uma barra de progresso (<Progress value={progress} />) enquanto o PDF carrega.

Mapeia o estado blocks para gerar os <Draggable>. O texto exibido no ecrã deve usar uma função parseVariables(block.text) que substitui visualmente {aluno} por "Nome de Exemplo", {curso} pelo nome real do curso selecionado, etc.

Aplica o estilo dinâmico no ecrã: fontWeight: block.bold ? 'bold' : 'normal', fontStyle: block.italic ? 'italic' : 'normal', fontFamily: block.font.

4. Otimização do Drag & Drop:

Usa o evento onStop do <Draggable> para atualizar as coordenadas X e Y no estado, evitando re-renderizações excessivas durante o arrastar (o que causa lentidão).

Garante que o CSS do bloco arrastável tem cursor: grab, bordas visíveis quando passa o rato (hover), e fundo ligeiramente translúcido para ser legível.

5. Lógica de Geração (pdf-lib) com Estilos e Fontes Diferentes:
Na função handleGenerate:

O pdf-lib precisa dos ficheiros .ttf físicos para criar os estilos. Transfere (fetch) as seguintes fontes do repositório raw do Google Fonts:

Alex Brush (Regular)

Poppins (Regular, Bold, Italic, BoldItalic)

Usa o @pdf-lib/fontkit para registar e incorporar (embedFont) todas estas 5 variações.

No loop de alunos, ao desenhar cada texto (drawText), deves aplicar a lógica para escolher a fonte correta com base no estado do bloco (ex: se block.font === 'Poppins' e block.bold e block.italic, usa a poppinsBoldItalicFont).

Antes de desenhar, substitui as variáveis na string final do aluno específico ({aluno} = student.fullName, {curso} = course.title, {data} = dataAtual, etc.).

Mantém o cálculo de inversão do eixo Y (yPdf = pdfHeight - (yDom * scaleFactor) - fontSize) para que o texto caia no local exato.
